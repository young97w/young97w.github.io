func search(nums []int, target int) int {  
    left, right := 0, len(nums)-1  
    for left <= right {  
       mid := left + (right-left)/2  
       if nums[mid] == target {  
          return mid  
       } else if nums[mid] < target {  
          left = mid + 1  
       } else {  
          right = mid - 1  
       }  
    }  
    return -1  
}

Q1: mid := (right + left) / 2 和 mid := left + (right - left) / 2 的区别

这是一个很好的问题。这两种计算中间索引的方法看起来很相似，但在某些情况下会有重要的区别。让我们来比较一下这两种方法：

  1. mid := (right + left) / 2
  2. mid := left + (right - left) / 2

主要区别：

  1. 整数溢出问题：

    • (right + left) / 2 可能会导致整数溢出。如果 right 和 left 都是非常大的正数，它们的和可能会超过整数类型能表示的最大值。
    • left + (right - left) / 2 不会导致溢出，因为它首先计算差值，然后进行除法，最后加上 left。

  2. 数学等价性：

    这两个表达式在数学上是等价的。我们可以通过代数来证明：

    1
2
3
4

    left + (right - left) / 2
= left + right/2 - left/2
= (2*left + right - left) / 2
= (left + right) / 2

  3. 性能：

    在大多数现代编译器和处理器上，这两种方法的性能差异可以忽略不计。

  4. 可读性：

    • (right + left) / 2 可能更直观，直接表达了"取两个数的平均值"的意图。
    • left + (right - left) / 2 虽然看起来稍微复杂一些，但它明确地表示了"从左边界开始，加上区间一半的长度"。

推荐使用 left + (right - left) / 2 的原因：

  1. 安全性：它避免了整数溢出的风险，使得代码在处理非常大的数组时也能正确工作。

  2. 通用性：这种写法在各种编程语言和整数类型中都能正确工作，不受限于特定的整数表示范围。

  3. 精确性：在某些编程语言中（如 Java），当 left 和 right 都是 int 类型时，(right + left) 的结果可能会在除以 2 之前溢出。而 left + (right - left) / 2 能确保中间结果始终在 int 范围内。

示例情况：

假设我们使用 32 位有符号整数，left = 2^30，right = 2^30 + 100：

  • (right + left) / 2 可能会导致溢出，因为 2^30 + 2^30 超过了 32 位整数能表示的最大值。
  • left + (right - left) / 2 会正确计算：2^30 + (100) / 2 = 2^30 + 50。

总结：虽然在大多数常见情况下，这两种方法都能正确工作，但使用 left + (right - left) / 2 是一种更安全、更通用的做法，特别是在处理可能涉及非常大的索引值时。

func removeElement(nums []int, val int) int {  
    // 处理空数组情况  
    if len(nums) == 0 {  
       return 0  
    }  
      
    // 双指针  
    left := 0  
    right := len(nums) - 1  
  
    // 当左指针小于等于右指针时继续  
    for left <= right {  
       // 找到左边等于val的元素  
       for left <= right && nums[left] != val {  
          left++  
       }  
  
       // 找到右边不等于val的元素  
       for left <= right && nums[right] == val {  
          right--  
       }  
  
       // 如果左指针仍然小于等于右指针，交换元素  
       if left <= right {  
          nums[left] = nums[right]  
          left++  
          right--  
       }  
    }  
  
    return left  
}

func removeElement(nums []int, val int) int {  
    slow , fast := 0,0  
    for fast < len(nums) {  
       if nums[fast] != val {  
          nums[slow] = nums[fast]  
          slow++  
       }  
       fast++  
    }  
  
    return slow  
}

func sortedSquares(nums []int) []int {  
    res := make([]int, len(nums))  
    index := len(nums) - 1  
    l, r := 0, len(nums)-1  
  
    for l <= r {  
       lv := nums[l] * nums[l]  
       rv := nums[r] * nums[r]  
       if lv < rv {  
          res[index] = rv  
          r--  
       } else {  
          res[index] = lv  
          l++  
       }  
       index--  
    }  
  
    return res  
}
